Format: Single compact block. Left accent border in brand primary color. संबंधित बातमी label. One headline with arrow link.

User is actively reading here. Keep it tightly relevant. Never use trending as a fallback for inline related.

First 3 cards deepen relevance. Slots 4-5 expand exploration to other cities. Slot 6 acts as a discovery hook. This prevents a repetitive single-topic feed and increases session depth naturally.

The TrendingModule below Related Stories is site-wide top 5. It serves as the final session-extension layer before the user exits the article.

05 Category and City Page Logic

These are the same page template. The route /category/[slug] serves both topic categories and city pages. No separate city route is needed.

Page sections in order

  1. HeroCard - top story for that category or city
  2. Latest feed - 6 stories, 2-col grid
  3. TrendingModule - top 5 within category
  4. City sub-sections (desktop only) - 3 cities x 1 story, for Maharashtra parent category
  5. Evergreen section - 4 long-form or reference stories
  6. Related Tags cloud
  7. Sidebar - Most Read (numbered) + Recently Updated + Ads

City page as dynamic content hub

/category/pune
  Hero: Top Pune story
  Latest: Latest Pune articles (all topics)
  Sections:
    Pune + Crime | Pune + Politics | Pune + Traffic | Pune + Civic
  (Dynamic filtered sections - NOT separate routes)

Filtering uses the city field combined with topic tags. Zero new routes are created.

06 Area Logic - Managed as Tags

Areas like Hinjawadi, Baner, Wakad, Kothrud are not separate categories and not separate schema fields. They are managed as geo-tags alongside topic tags on the article.

Why Tags, not a separate field

  • Same tagging interface editors already use
  • Automatically powers a /tag/hinjawadi page if article volume justifies it
  • Feeds the recommendation engine without schema complexity
  • No new routes, no new CMS fields, no new UI

Example article tagging

Category: Pune
Topic: Crime
Tags: #हिंजवडी, #पुणे पोलीस, #सायबर गुन्हे

Recommendations this article feeds

  • Pune Crime stories (city + topic match)
  • Other Hinjawadi tag stories (area match)
  • Maharashtra Crime stories (category match)
  • Pune Police tag stories (tag overlap)
Area tag creation rule: Create the area tag only when it will have 5+ articles per month consistently. Do not create tags for one-off event locations.

07 Tag Pages

URL: /tag/[slug]

Page sections: Hero, Latest feed, TrendingModule, Related Tags cloud, Sidebar.

Tag pages serve as topic hubs - examples: #मराठा आरक्षण, #मेट्रो, #हिंजवडी.

Global tag creation rule: Only create a tag if it will have 10+ articles per month in sustained volume. Avoid duplicate, random, or low-volume tags.

09 Architecture Goals

GoalHow it is achieved
Clean SEOCategory and City share one route template. No URL sprawl.
Scalable citiesCity pages are filtered views, not separate category trees.
Scalable areasAreas are Tags. No new routes or fields needed.
Editorial controlCMS labels override the algorithm at any point.
Session depthInline related, then end-of-article related, then Trending - 3 retention layers.
Topical authorityTags cluster articles for Google Discover and topic-based crawling.
Developer simplicity6 page templates cover the entire site. One recommendation engine serves all.
Dev implementation note: Recommendation engine requires article fields: category, topic, city, tags[], breakingStatus, publishedAt, trafficVelocity. The only new addition to the current schema is the explicit city field. Areas are fully covered by the existing tags system.
